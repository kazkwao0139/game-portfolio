PDA 스노볼 & 교착 감지 함수

수학적 모델 정의서

1. 팀 전투력 함수

\[ P(t) = w_L \cdot L(t) + w_N \cdot N(t) + w_X \cdot X(t) + w_C \cdot C(t) \]

기호 정의 범위
\(P(t)\) 시간 \(t\)에서의 팀 전투력 \(\mathbb{R}^+\)
\(L(t)\) 공유 팀 레벨 \([1, L_{\max}]\)
\(N(t)\) 노드 점령 가치: \(\displaystyle\sum_{i \in \text{점령중}} v_i\) \([0, N_{\max}]\)
\(X(t)\) 팀 무기 XP 총량 \(\mathbb{R}^+\)
\(C(t)\) 정보 장악률 (CCTV 점유율) \([0, 1]\)
\(w_i\) 가중치 계수, \(\displaystyle\sum_i w_i = 1\) \([0, 1]\)

노드 가치 (고정값)

\[ v_i = \begin{cases} 1 & \text{1차 앵커} \\ 2 & \text{2차 앵커} \\ 5 & \text{3차 앵커} \end{cases} \]

※ 위 수치는 개념 설명을 위한 예시이며, 실제 값은 밸런싱 과정에서 조정됩니다.

2. XP 소스 분리

\[ X(t) = X_{\text{node}}(t) + X_{\text{downlink}}(t) \]

소스 정의 상한 특성
\(X_{\text{node}}(t)\) 노드 점령으로 얻는 XP \(X_{\text{node}}^{\max}\) 한정
\(X_{\text{downlink}}(t)\) 다운(킬)으로 얻는 XP \(\infty\) (획득 기회) 아래 참조

다운링크 XP의 특성

  1. 1v1 다운 > 4v1 다운 (기여도 가중 배분)
  2. 킬당 XP는 고정값 (XP 자체가 늘어나지 않음)
  3. 획득 기회는 무한 (영토/CCTV/노드XP는 상한 존재)
  4. 약한 피드백: XP 많으면 → 다음 전투 유리 → 다음 XP 얻기 쉬움

다운링크 XP 배분 모델

설계 의도: 솔로 캐리가 불가능하다는 우려를 인지하고 있습니다. 그래서 솔로 다운링크의 가치가 크다고 설계했습니다.

모델 1: 균등 배분 \[ XP_{팀} = XP_{base} \times \frac{1}{n} \]

n XP
1 100%
4 25%

직관적. 플레이어 체감 좋습니다.

모델 2: 란체스터 기반 \[ XP_{팀} = XP_{base} \times \frac{1}{n^2} \]

n XP
1 100%
4 6.25%

수학적 일관성. 리스크-리워드 연동입니다.

모델 3: 하이브리드 (가중 평균) \[ XP_{팀} = XP_{base} \times \left( \alpha \cdot \frac{1}{n} + (1-\alpha) \cdot \frac{1}{n^2} \right) \]

n α=1.0 α=0.7 α=0.5 α=0.0
1 100% 100% 100% 100%
2 50% 42.5% 37.5% 25%
4 25% 19.7% 15.6% 6.25%

\[ \boxed{\text{구조는 고정, 수치(α)는 튜닝}} \]

주: \(n\)은 킬에 참여한 아군 인원수. \(XP_{base}\)는 1v1 킬 시 팀이 받는 기본 XP. 팀 레벨은 공유이므로, 획득한 \(XP_{팀}\)은 팀원 전체에게 동일하게 적용됩니다.

저자 주: 수학적 엄밀함을 택해야 할지, 현실과 타협해야 할지 고민하다가 일단 세 모델 다 적는다. 하이브리드 모델의 α는 데이터팀과 밸런스팀이 멋지게 튜닝해줄 거라 믿습니다.

물론 솔로 다운링크를 미친 듯이 터뜨리는 플레이어가 있을 수 있습니다. 그렇게 되면 초선형이 아니라 지수함수형으로 스노볼이 폭주하기 시작하지만, 그건 그저 그 플레이어가 지나치게 잘해서 생기는 일이지 시스템의 문제는 아니라고 생각합니다.

3. 자원의 한정성 정리

자원 상한 특성
\(L(t)\) \(L_{\max} = 5\) 한정
\(N(t)\) \(N_{\max}\) = 맵 노드 총합 한정
\(C(t)\) \(C_{\max} = 1\) 한정
\(X_{\text{node}}(t)\) \(X_{\text{node}}^{\max}\) 한정
\(X_{\text{downlink}}(t)\) \(\infty\) (획득) 단, 효과는 \(L_{\max}\)에서 소멸

핵심 구분: 획득 vs 효과

\[ \boxed{\text{XP 획득} \neq \text{전투력 전환}} \]

구분 상한
다운링크 XP 획득 무한 (킬 횟수 제한 없음)
다운링크 XP 효과 (→ 레벨) \(L_{\max} = 5\)

레벨 5 도달 이후: - XP 계속 획득 가능 - 그러나 \(L(t) = L_{\max}\) → 레벨업 없음 - 전투력 \(P(t)\)에 기여하는 \(w_L \cdot L(t)\) 항 고정 - XP의 전투력 전환 효과 소멸

4. 격차 함수 (스노볼 지표)

\[ G(t) = P_A(t) - P_B(t) \]

5. 모델 버전 1: 순수 선형 (이론적 모델)

다운링크 XP의 피드백 효과를 무시한 이상적 모델

가정

  • 다운링크 XP가 다음 전투에 미치는 영향 무시
  • 또는 \(w_X \approx 0\)

미분방정식

\[ \frac{dG}{dt} = k \quad (k = \text{상수}) \]

해

\[ G(t) = G_0 + kt \]

역전 소요 시간

\[ t_{\text{역전}} = -\frac{G_0}{k} \]

특성

  • 완전 선형: 격차가 격차를 가속하지 않음
  • 역전 항상 가능: 유한 시간 내 따라잡기 가능
  • 예측 가능: 게임 흐름이 계산 가능

6. 모델 버전 2: 약한 초선형 (실제 모델)

다운링크 XP가 다음 다운을 약간 쉽게 만드는 효과 포함

연립 미분방정식

\[ \begin{cases} \displaystyle\frac{dG}{dt} = k + \mu \cdot X_{\text{downlink}}(t), \quad \mu \ll 1 \\[1em] \displaystyle\frac{dX_{\text{downlink}}}{dt} = h(G, t) \end{cases} \]

파라미터 의미
\(k\) 기본 격차 변화율
\(\mu\) 다운링크 피드백 계수 (작은 값)
\(h(G, t)\) 격차와 시간에 따른 다운링크 XP 획득 함수

특성

  • 약한 초선형: 피드백 있지만 지수적 폭주 아님
  • \(w_X\)가 작으면 선형에 근사
  • 캐리 판타지 일부 보존: 킬 많으면 약간 더 강해짐

수치 해석

  • 연립 상미분방정식으로 표현됨
  • Runge-Kutta, Euler 등 수치 해석으로 실시간 계산 가능
  • 밸런싱 시뮬레이션으로 \(\mu\), \(k\), \(w_X\) 최적값 탐색 가능

6.1 분기점(Bifurcation Point) 불성립 논증

예상 질문: “\(\mu\)가 임계값을 넘으면 지수적 폭주로 전환되는 분기점이 존재합니까?”

답변: 분기점 논의 자체가 성립하지 않습니다.

\(\mu\)가 의미를 가지려면 XP → 전투력 전환이 지속적으로 일어나야 합니다.

\[ \mu \cdot X_{\text{downlink}} \xrightarrow{\text{효과}} w_L \cdot \Delta L \]

그러나:

\[ L(t) \leq L_{\max} = 5 \]

레벨 5 도달 시:

\[ \Delta L = 0 \implies \mu \cdot X_{\text{downlink}} \xrightarrow{\text{효과}} 0 \]

조건 \(\mu\)의 효과
\(L < L_{\max}\) 존재 (약한 피드백)
\(L = L_{\max}\) 소멸

\[ \boxed{\mu \text{가 아무리 커도, } L_{\max} \text{라는 경계 조건이 효과를 차단합니다.}} \]

결론:

분기점(Bifurcation)은 시스템이 무한히 발산할 수 있을 때 의미가 있습니다. PDA는 \(L_{\max}\)라는 흡수 경계(Absorbing Boundary)가 존재하므로, \(\mu\)의 크기와 무관하게 지수적 폭주는 구조적으로 불가능합니다.

7. 경계 조건에 의한 상한 (후반 게임 역학)

선형 증가의 한계

격차 함수 \(G(t)\)는 선형으로 증가하려 하지만:

\[ G(t) = G_0 + kt \quad \text{(무한 증가?)} \]

아니다. 획득 가능한 자원에 상한이 존재한다:

\[ N(t) \leq N_{\max}, \quad L(t) \leq L_{\max}, \quad C(t) \leq 1 \]

한 팀이 완전히 몰아붙였을 때

한 팀이 상대 영토를 전부 점령하면:

\[ N_A(t) \to N_{\max}, \quad N_B(t) \to 0 \]

이 상태에서: - 더 이상 점령할 노드가 없음 - 노드 XP 획득 불가 - 다운링크 XP만 남음

XP 획득의 변화

\[ \frac{dX}{dt} = \frac{dX_{\text{node}}}{dt} + \frac{dX_{\text{downlink}}}{dt} \to \frac{dX_{\text{downlink}}}{dt} \]

상대 영토를 전부 먹으면 노드 XP는 더 이상 획득할 수 없고, 다운링크만 남는다.

다운링크 XP의 이중 경계

  1. 영토 경계: 노드 전부 점령 시 → 노드 XP 소멸
  2. 레벨 경계: \(L = L_{\max}\) 시 → XP 효과 소멸

\[ \boxed{\text{다운링크 XP는 "유일한 무한 소스"이지만, 두 경계 조건에 의해 효과가 소멸됩니다.}} \]

격차 변화율의 감소

\[ \frac{dG}{dt} \to w_X \cdot \frac{dX_{\text{downlink}}}{dt} \]

\(w_X\)가 작으면:

\[ \frac{dG}{dt} \to 0 \quad \text{(상한 도달)} \]

수학적 명확화

\[ \boxed{\text{선형 증가(Linear Growth) + 경계 조건(Boundary Condition) = 상한 도달(Ceiling)}} \]

이는 수렴(Convergence)이 아니라, 구간별 선형(Piecewise Linear)이되 경계에서 멈추는 구조다.

그래프

G(t)
  │
  │                    ─────────────── 상한 (L = L_max, N = N_max)
  │               ─────
  │          ─────
  │      ────
  │   ───
  │ ──
  └──────────────────────────────────── t
      초반      중반      후반
      (노드+다운링크)   (다운링크만, 효과 감소)

게임 흐름

단계 상태 XP 소스 스노볼 속도
초반 영토 경쟁 노드 XP + 다운링크 XP 높음
중반 밀어붙이기 노드 비중 감소 감소
후반 레벨 상한 도달 다운링크만 (효과 소멸) \(\approx 0\)
결정전 교착 트리거 브레이커 스폰 강제 분기

8. 교착 상태 감지

정의

\[ \text{교착}(t, \tau) := \left| \frac{dG}{dt} \right| < \epsilon \quad \forall \ t' \in [t-\tau, \ t] \]

파라미터 의미 예시
\(\tau\) 관측 윈도우 60초
\(\epsilon\) 임계값 0.01

트리거 조건

\[ \boxed{\text{교착}(t, \tau) = \texttt{true} \implies \text{오브젝트 스폰}} \]

해석: “최근 \(\tau\)초 동안 양 팀의 격차가 벌어지지도, 좁혀지지도 않았다”

9. 오브젝트 스폰 규칙

반응-확산 방정식과 란체스터 법칙 기반

1. 개요

PDA에서 채널링 시간은 “감”으로 정하는 상수가 아니라, 시스템 변수들의 함수로 도출되는 해(Solution)입니다.

본 문서는 다음 질문에 답한다:

“3v4 교전 리스크, 노드 가치, 맵 구조를 고려했을 때, 균형 잡힌 채널링 시간은 몇 초인가?”

2. 변수 정의

2.1 맵/챔피언 변수 (고정)

기호 정의 단위
\(d\) 노드 간 평균 거리 m
\(v\) 챔피언 평균 이동속도 m/s
\(r\) 챔피언 평균 스킬 사거리 m
\(d_{spawn}\) 리스폰 지점 → 노드 평균 거리 m
\(\bar{a}\) 챔피언 평균 공격력 -

저자 주 (방어력에 대하여): 방어력(Resistance)에 대해선 당장 답을 찾지 못했습니다. 이 문서를 보는 여러분이라면 찾을 거라 믿습니다. 우선은 제외하였습니다.

주: 스킬 계수와 대미지 공식의 추상화

본 문서의 변수들은 개별 스킬을 추상화한 값이다:

변수 실제 포함 요소
\(v\) (이동속도) 기본 이동속도 + 무빙기(대시, 블링크, 가속 등) 포함 평균
\(\bar{a}\) (공격력) 기본 공격력 + 스킬 계수, 스킬 대미지 포함 평균
\(r\) (사거리) 기본 사거리 + 스킬 사거리 포함 평균

즉, 개별 챔피언의 스킬셋 전체를 단일 수치로 압축한 것입니다. 실제 밸런싱 시에는 이 추상화된 값을 기준으로 개별 스킬 계수를 역산합니다.

2.2 게임 상태 변수

기호 정의 범위 용도
\(A\) 아군 교전 인원 \([1, 4]\) 전투력 계산 (플레이어 참고용)
\(B\) 적군 교전 인원 \([1, 4]\) 전투력 계산 (플레이어 참고용)
\(L\) 팀 레벨 \([1, 5]\) 보호막 결정
\(S(L)\) 채널링 보호막 (레벨 비례) \(\mathbb{R}^+\) 채널링 가능 여부
\(t_{respawn}\) 리스폰 대기 시간 레벨 기반 채널링 윈도우
\(w_i\) 노드 \(i\)의 밸류 (위치/상태 포함) \(\mathbb{R}^+\) 채널링 시간 결정

2.3 도출 변수

기호 정의
\(T^*\) 균형 채널링 시간 (목표 해)

3. 적 도착 시간 모델

3.1 기본 도착 시간

적이 인접 노드에서 방해하러 오는 시간:

\[ t_{arrive} = \frac{d - r}{v} \]

항 의미
\(d\) 노드 간 거리
\(r\) 스킬 사거리 (도착 전에 끊을 수 있음)
\(v\) 이동속도

3.2 확률적 도착 모델

적이 시간 \(t\) 내에 도착할 확률 (지수 분포 가정):

\[ P(t_{arrive} < t) = 1 - e^{-\lambda t} \]

여기서 도착률 \(\lambda\):

\[ \lambda = \frac{v}{d} \cdot n_{nearby} \]

항 의미
\(v/d\) 이동속도/거리 비율
\(n_{nearby}\) 인접 노드의 적 수

4. 란체스터 제곱 법칙 (수정)

4.1 기본 공식

전투력은 인원수와 공격력/방어력의 함수:

\[ \text{전투력} = n^2 \times \frac{a}{h} \]

항 의미
\(n\) 인원수
\(a\) 공격력
\(h\) 방어력(체력)

4.2 채널링 보호막

채널링 중인 챔피언에게 레벨 비례 보호막 부여:

\[ S_{mean} = S(L) = s_0 + s_1 \cdot L \]

\[ S_{champ} = S_{mean} \times k_{champ} \]

파라미터 의미
\(S_{mean}\) 평균 보호막 (레벨 기반)
\(s_0\) 기본 보호막
\(s_1\) 레벨당 보호막 증가량
\(L\) 팀 레벨
\(S_{champ}\) 챔피언별 보호막
\(k_{champ}\) 챔피언별 보호막 계수 (이동속도에 반비례)

왜 공격력이 아니라 보호막인가?

해킹 중에는 공격을 하지 않습니다. 따라서 이동속도가 빠른 챔피언의 공격력을 깎아봤자 해킹 성공률에는 영향이 없습니다.

  • 빠른 챔피언: 해킹 시도 기회(Frequency)가 많음
  • 밸런싱: 기댓값을 맞추려면 성공 확률(Probability)을 낮춰야 함
  • 방법: 보호막(S)을 줄여서, 스치기만 해도 해킹이 취소되게 함

결론: 공격력을 깎는 건 '전투' 밸런스지만, 보호막을 깎는 건 '승리' 밸런스입니다.

챔피언별 보호막 계수 예시

챔피언 이동속도 \(k_{champ}\) 특성
밤 (Bam) Fast 0.6 물몸 — 스치면 끊김
별 (Byul) Normal 1.0 기준점
산 (San) Slow 1.4 철옹성 — 짤짤이 무시

자동 밸런싱: 신규 챔피언의 이동속도만 입력하면, 보호막 계수가 자동으로 도출됩니다.

공격력(DPS)의 의미

공격력은 "체력을 깎는 힘"이 아니라, "상대의 해킹 시도를 무산시키는 저지력(Stopping Power)"입니다.

\[ t_{shield} = \frac{S_{champ}}{DPS} \]

채널링 성공 조건: \(T^* < t_{shield}\) (보호막이 채널링 완료까지 버텨야 함)

중요: 보호막은 레벨에만 비례. 노드 가치와 무관합니다.

4.3 채널링 취소 조건

\[ \text{채널링 취소} \iff \begin{cases} S(L) \leq 0 & \text{(보호막 파괴)} \\ \text{CC hit} = \texttt{true} & \text{(CC기 적중)} \end{cases} \]

조건 결과
보호막 깨짐 채널링 즉시 취소
CC기 적중 채널링 즉시 취소
일반 공격 보호막 깎임, 채널링 유지

4.4 전투 구조 (채널링 중)

[적 B명] ←→ [아군 전투원 (A-1)명] ←― [채널러 1명 (후방)]
                    ↑                        ↑
               전선에서 막아줌          보호막으로 버팀

적의 선택지: 1. 전선 교전 → \((A-1)\) vs \(B\) 2. 채널러 암살 시도 → 보호막 깨거나 CC로 끊기

4.5 유효 전투력 비

전선 교전 시:

\[ \rho = \frac{(A-1)^2}{B^2} \]

상황 전투원 \(\rho\)
4v4 정상 4 vs 4 1.00
채널링 (3v4) 3 vs 4 0.56
채널링 (2v3) 2 vs 3 0.44
채널링 (1v2) 1 vs 2 0.25

중요: 이 전투력 비는 플레이어의 리스크 판단용입니다. \[ \boxed{\text{채널링 시간 } T^* \text{는 } \rho \text{에 영향받지 않는다 (고정값).}} \]

“지금 채널링해도 될까?”는 플레이어가 \(\rho\)를 보고 스스로 판단합니다.

4.6 보호막 생존 시간

적이 채널러를 공격할 때 보호막이 버티는 시간:

\[ t_{shield} = \frac{S(L)}{n_{attacker} \times \bar{a}} \]

채널링 성공 조건: \[ T < t_{shield} \quad \text{(보호막이 채널링 완료까지 버텨야 함)} \]

또는 아군이 적 암살자를 저지해야 합니다.

5. 채널링 윈도우 (팀파이트 후)

5.1 안전 윈도우

적 전멸 후 채널링 가능 시간:

\[ t_{window} = t_{respawn} + \frac{d_{spawn}}{v} \]

항 의미
\(t_{respawn}\) 적 리스폰 대기 시간
\(d_{spawn}/v\) 리스폰 → 노드 이동 시간

5.2 안전 채널링 조건

\[ T < t_{window} \implies \text{방해 없이 채널링 완료} \]

6. 균형 채널링 시간 도출

6.1 핵심 원칙

요소 비례 대상 역할
보호막 \(S(L)\) 레벨만 채널링 “가능 여부” (방어)
채널링 시간 \(T\) 노드 가치 리스크 (높을수록 오래 걸림)

6.2 최종 공식

\[ T^* = \alpha \cdot t_{arrive} + \gamma \cdot w_i \]

항 의미 특성
\(\alpha \cdot t_{arrive}\) 적 도착 시간 반영 맵 구조에서 도출 (고정)
\(\gamma \cdot w_i\) 노드 가치 반영 노드별 고정값

\(\alpha, \gamma > 0\): 튜닝 계수

6.3 보호막 제약 조건

채널링이 성공하려면 보호막이 버텨야 함:

\[ T^* < t_{shield} = \frac{S(L)}{n_{attacker} \times \bar{a}} \]

또는 아군이 채널러를 보호해야 합니다.

6.4 해석

\(t_{arrive}\) 항: - 적이 멀리 있는 노드 → 채널링 길어도 됨 - 적이 가까운 노드 → 채널링 짧아야 함

\(w_i\) 항: - 3차 앵커 (고가치) → 리스크 높여야 하므로 T 증가 - 1차 앵커 (저가치) → 쉽게 먹을 수 있어야 하므로 T 감소

6.5 중요: 채널링 시간은 고정값

\[ \boxed{\text{채널링 시간은 교전 상황(ρ)에 따라 변하지 않는다.}} \]

변수 채널링 시간에 영향
맵 구조 (\(t_{arrive}\)) ✓
노드 가치 (\(w_i\)) ✓
교전 인원 (\(\rho\)) ✗

13. 신규 챔피언 밸런싱 예측

13.1 핵심 인사이트

PDA의 모든 수치는 서로 방정식의 해가 맞물리면서 도출됩니다:

\[ v \rightarrow k_{champ} \rightarrow S_{champ} \rightarrow DPS_{required} \]

  • 이동속도(\(v\))가 보호막 계수(\(k_{champ}\))를 결정하고,
  • 보호막(\(S\))이 공격력(\(DPS\))의 기준을 결정합니다.

이 톱니바퀴 구조 덕분에, 어느 하나만 툭 튀어나온 OP(Over Powered)가 나올 수 없습니다.

균형 채널링 시간 \(T^*\)는 평균 챔피언 능력치를 기반으로 도출됩니다:

\[ T^* = f(\bar{v}, \bar{r}, d, w_i) \]

따라서 신규 챔피언의 개별 능력치가 평균에서 얼마나 벗어나는지 계산하면, 출시 전에 밸런스 계수를 자동으로 도출할 수 있습니다.

13.2 신규 챔피언 계수 도출

신규 챔피언의 이동속도 \(v_{new}\)만 입력하면, 나머지는 자동으로 도출됩니다:

\[ k_{champ} = \frac{\bar{v}}{v_{new}} \]

\[ S_{champ} = S_{mean} \times k_{champ} \]

입력 자동 도출
\(v_{new} > \bar{v}\) (빠름) \(k_{champ} < 1\) → 보호막 감소
\(v_{new} = \bar{v}\) (평균) \(k_{champ} = 1\) → 기준점
\(v_{new} < \bar{v}\) (느림) \(k_{champ} > 1\) → 보호막 증가

기획자가 할 일: 이동속도만 정하면 끝. 보호막은 방정식이 알아서 맞춥니다.

13.3 추상화 → 개별 스킬 역산

균형 방정식에서 도출된 추상화된 평균값을 기준으로, 개별 스킬 계수를 역산한다:

\[ \bar{v}_{new} = v_{base} + \sum_{i} w_i \cdot v_{skill_i} \]

단계 작업
1 균형 방정식에서 \(\bar{v}_{new}\) 목표치 도출
2 기본 이동속도 \(v_{base}\) 설정
3 나머지를 무빙기 스킬들에 배분

예시: - 목표 \(\bar{v}_{new} = 420\) - 기본 이동속도 \(v_{base} = 350\) - 대시 스킬 기여분 = \(420 - 350 = 70\) - → 대시 스킬의 거리/쿨다운 조합으로 70에 맞춤

13.4 기존 방식 vs PDA 방식

기존 게임 PDA
출시 전 감으로 스펙 설정 공식에 대입
출시 후 핫픽스, 밸런스 패치 반복 예측 검증
밸런스 주기 2주~1달 출시 전 확정 가능
예측 가능성 낮음 높음

13.5 결론

\[ \boxed{v \rightarrow k_{champ} \rightarrow S_{champ} \rightarrow DPS_{required}} \]

15. 반응-확산 방정식과의 연결

본 모델은 아래의 반응-확산 시스템을 간소화한 것이다:

\[ \frac{\partial U}{\partial t} = D \nabla^2 U - \vec{v} \cdot \nabla U + R(U) \]

PDE 항 본 모델에서의 대응
\(D \nabla^2 U\) (확산) \(t_{arrive}\): 맵 구조에서 도출되는 도착 시간
\(\vec{v} \cdot \nabla U\) (이류) \(v\): 챔피언 이동속도
\(R(U)\) (반응) \(w_i\): 노드 가치 (리스크 결정)

정상 상태(\(\frac{\partial U}{\partial t} = 0\))에서의 해가 바로 균형 채널링 시간 \(T^*\)입니다.

보호막의 역할

보호막 \(S(L)\)은 PDE의 경계 조건(Boundary Condition)에 해당:

  • \(S(L) > 0\): 채널링 유지 가능 (경계 안정)
  • \(S(L) \leq 0\) 또는 CC 적중: 채널링 취소 (경계 붕괴)

란체스터 법칙의 역할

란체스터 법칙(\(\rho = (A-n)^2 / B^2\))은 두 가지 용도로 사용된다:

  1. 단일 채널링: 플레이어가 “지금 채널링해도 되는가?”를 판단할 때 참고하는 리스크 지표
  2. 다중 채널링: 채널링 시간 단축 비율 \(T^*_n / T^*\)를 결정하는 보상 계수

\[ \boxed{\text{단일 채널링: } T^* \text{는 고정값 (ρ 무관)}} \] \[ \boxed{\text{다중 채널링: } T^*_n = T^* \cdot \frac{\rho_n}{\rho_1} \text{ (ρ 연동)}} \]

본 문서의 모델은 간소화되어 있습니다. 시야(CCTV)는 복잡도를 줄이기 위해 제외되었고, 개별 챔피언 스킬 사거리는 평균값으로 처리되었습니다. 실제로는 더 복잡한 요소가 개입할 수 있습니다.

본 문서의 모든 방정식, 파라미터, 수치는 예시에 불과합니다. 더 정확한 방정식과 수치는 여러분이 더 잘 찾을 것이라고 확신합니다.

PDA 채널링 시간 공식의 유도

차원 해석(Dimensional Analysis)을 통한 특성 시간 도출

Q. 정상 상태 조건에서 선형 공식이 어떻게 도출되는가?

“\(\frac{\partial U}{\partial t} = 0\) 조건에서 어떻게 \(T^* = \alpha \cdot t_{arrive} + \gamma \cdot w_i\)가 나오는가?”

A. 답변

해당 공식은 편미분방정식을 해석적으로 엄밀하게 푼 결과(Exact Solution)가 아니라, ‘차원 해석(Dimensional Analysis)’을 통해 도출된 특성 시간(Characteristic Time)의 선형 결합입니다.

복잡한 비선형 항을 소거하고, 게임 디자인에 필요한 지배 방정식(Governing Equation)의 주요 항만을 추출하여 근사하였습니다.

1. 지배 방정식 정의 (1차원 단순화)

전장의 한 레인(Circuit)을 1차원 공간 \(x\)로 가정하고, 영향력 밀도 \(U\)에 대한 반응-확산 방정식을 씁니다.

\[ \frac{\partial U}{\partial t} = D \frac{\partial^2 U}{\partial x^2} - v \frac{\partial U}{\partial x} - kU + S \]

항 기호 물리적 의미
확산 \(D \frac{\partial^2 U}{\partial x^2}\) 정보/영향력의 자연 전파
이류 \(v \frac{\partial U}{\partial x}\) 챔피언의 이동 (Advection)
반응 \(kU\) 노드의 저항성 (Reaction/Decay)
소스 \(S\) 해킹 시도에 의한 영향력 주입

2. 정상 상태(Steady State) 가정

채널링이 완료되는 시점은 시스템이 평형에 도달하는 순간이므로, 시간 변화율을 0으로 둡니다.

\[ \frac{\partial U}{\partial t} = 0 \]

\[ 0 = D \frac{d^2 U}{d x^2} - v \frac{d U}{d x} - kU + S \]

3. 스케일링(Scaling) 및 차원 해석

우리는 \(U(x)\)의 곡선 모양이 아니라, 이 과정이 완료되는 데 필요한 ‘시간 척도(Time Scale)’가 궁금한 것입니다.

미분 연산자를 특성 길이 \(L\) (노드 간 거리)로 치환하여 차원(Dimension)만 분석합니다.

\[ \frac{d U}{d x} \sim \frac{U}{L}, \quad \frac{d^2 U}{d x^2} \sim \frac{U}{L^2} \]

이를 정상 상태 식에 대입하여 시간 차원(\(T\))을 역산합니다.

3.A 이류(Advection) 항의 시간 척도

이류 항 \(v \frac{\partial U}{\partial x}\)이 지배적일 때, 특성 시간 \(\tau_{adv}\)는:

\[ \tau_{adv} \sim \frac{L}{v} \]

이것은 게임 내 변수로 \(t_{arrive}\) (적 도착 시간)에 해당합니다.

\[ t_{arrive} = \frac{d - r}{v} \]

변수 의미
\(d\) 적 스폰~앵커 거리
\(r\) 적 스킬 사거리
\(v\) 적 이동속도

3.B 반응(Reaction) 항의 시간 척도

반응 항 \(kU\) (노드 저항)이 지배적일 때, 특성 시간 \(\tau_{react}\)는 반응률의 역수입니다.

\[ \tau_{react} \sim \frac{1}{k} \]

여기서 반응 계수 \(k\)는 노드의 방어력이나 중요도에 반비례하므로, 노드 가치 \(w_i\)에 비례한다고 모델링합니다.

\[ \tau_{react} \propto w_i \]

노드 가치 \(w_i\)
1차 앵커 1
2차 앵커 2
3차 앵커 5

4. 선형 결합 (Linear Combination)

전체 채널링 시간 \(T^*\)는 이 두 가지 물리적 과정이 복합적으로 작용하는 총 시간입니다:

  1. 적이 도착하는 물리적 시간 (이류)
  2. 노드를 해킹하는 논리적 시간 (반응)

따라서, 두 특성 시간의 선형 결합으로 근사할 수 있습니다.

\[ T^* \approx C_1 \cdot \tau_{adv} + C_2 \cdot \tau_{react} \]

이를 게임 디자인 변수로 치환하면 최종 공식이 됩니다:

\[ \boxed{T^* = \alpha \cdot t_{arrive} + \gamma \cdot w_i} \]

5. 계수 해석

계수 정의 물리적 의미
\(\alpha\) 이류 항 가중치 맵 크기 및 기동성 민감도
\(\gamma\) 반응 항 가중치 노드 중요도 민감도

6. 차원 검증

\[ [T^*] = [\alpha] \cdot [t_{arrive}] + [\gamma] \cdot [w_i] \]

\[ [s] = [1] \cdot [s] + [s] \cdot [1] = [s] \quad \checkmark \]

7. 방법론적 정당성

차원 해석(Dimensional Analysis)은 물리학에서 표준적인 근사 방법론입니다.

예시 유도 방식
레이놀즈 수 (Reynolds Number) 관성력/점성력 비율의 차원 해석
마하 수 (Mach Number) 유속/음속 비율의 차원 해석
페클렛 수 (Péclet Number) 이류/확산 비율의 차원 해석

PDA의 채널링 시간 공식도 동일한 원리:

\[ \boxed{\text{이류 시간 척도} + \text{반응 시간 척도} = \text{총 채널링 시간}} \]

8. 요약

\[ \boxed{ \begin{aligned} &\text{반응-확산 PDE} \\ &\quad \downarrow \text{ 정상 상태 조건 } (\partial U/\partial t = 0) \\ &\text{지배 방정식의 주요 항 추출} \\ &\quad \downarrow \text{ 차원 해석} \\ &\text{두 특성 시간 도출: } \tau_{adv}, \ \tau_{react} \\ &\quad \downarrow \text{ 선형 결합} \\ &\boxed{T^* = \alpha \cdot t_{arrive} + \gamma \cdot w_i} \end{aligned} } \]

본 유도는 해석적 엄밀해가 아닌, 차원 해석을 통한 근사입니다. 레이놀즈 수(Reynolds Number) 유도와 같은 방법론을 사용했습니다. 계수 \(\alpha\), \(\gamma\)는 시뮬레이션과 플레이테스트를 통해 결정됩니다.

더 엄밀한 수학적 전개는 전문가분들의 검토와 보완을 기대합니다.

6. 반응-확산 PDE 기반 밸런싱

6.1 PDE와 대수방정식의 관계

채널링 시간 공식:

\[ T^* = \alpha \cdot t_{arrive} + \gamma \cdot w_i \]

주의: 이 식 자체는 대수방정식(Algebraic Equation)입니다. 미분 기호(\(\partial\))가 없습니다.

그러나 이 식은 반응-확산 PDE의 정상 상태 해(Steady State Solution)를 근사한 것이다:

\[ \frac{\partial U}{\partial t} = D \nabla^2 U - \vec{v} \cdot \nabla U + R(U) \]

정상 상태 조건(\(\frac{\partial U}{\partial t} = 0\))에서 위 PDE를 풀면, 균형점이 대수적 관계로 도출됩니다.

\[ \boxed{\text{원리: PDE} \to \text{밸런싱에 사용: 정상 상태 해 (대수식)}} \]

6.2 챔피언 스탯의 출처

PDA의 모든 챔피언은 동일한 균형 방정식에서 파라미터가 도출된다:

\[ t_{arrive} = \frac{d - \bar{r}}{\bar{v}} \]

평균 챔피언 능력치 \((\bar{v}, \bar{r}, \bar{a})\)가 공식에 포합니다.

저자 주: 방어력(Resistance)에 대해선 당장 답을 찾지 못했습니다. 이 문서를 보는 여러분이라면 찾을 거라 믿습니다. 우선은 제외하였습니다.

6.3 신규 챔피언 밸런싱

신규 챔피언의 편차: \[ \Delta v = v_{new} - \bar{v}, \quad \Delta r = r_{new} - \bar{r}, \quad ... \]

이 편차를 기반으로 출시 전 보정 계수 도출 가능.

6.4 왜 1티어가 생기기 어려운가

\[ \boxed{\text{모든 챔피언} \approx \text{균형점 근처 (정상 상태 해 기반)}} \]

2.2 핵심 인사이트

\[ \boxed{\text{서프라이즈는 끝내는 장치입니다. 서스펜스가 본질입니다.}} \]

화려한 콤보, 펜타킬, 폭발 이펙트 — 이것들은 전부 서프라이즈.

서스펜스 없이 서프라이즈만 연속되면?

터지고 → 끝 → 기다림 → 터지고 → 끝 → 기다림

공허한 자극의 반복.

2.3 미장센 (Mise-en-scène)

영화에서 미장센은 화면 구성과 시각적 연출을 의미합니다.

PDA에서 색깔이 이 역할을 한다:

요소 역할
노드 색깔 점령 상태를 한눈에 전달
맵 전체 색 분포 “누가 이기고 있나”를 시각적으로 표현
색의 변화 전세 역전을 직관적으로 보여줌

\[ \boxed{\text{색깔 = 미장센 = 서사의 시각화}} \]

2.4 맥거핀 (MacGuffin)

히치콕의 정의:

“캐릭터들은 신경 쓰지만, 관객에게는 그 자체로 중요하지 않은 것. 플롯을 이끄는 장치.”

PDA의 오브젝트가 맥거핀 역할을 한다:

오브젝트 맥거핀 역할
앵커 “저거 먹어야 해” → 행동 유발
토이 솔져 “저거 먹어야 해” → 행동 유발
플라스틱 드래곤 “저거 먹어야 해” → 행동 유발
디멘션 브레이커 “저거 먹어야 해” → 행동 유발

\[ \boxed{\text{오브젝트 = 맥거핀 = 플롯을 이끄는 장치}} \]

오브젝트 자체가 중요한 게 아닙니다. 오브젝트를 두고 벌이는 싸움이 긴장감을 만듭니다.

2.5 서스펜스의 구성 요소

서스펜스는 일반적으로 다음과 같은 요소들을 가진다. PDA는 이 요소들을 게임 시스템으로 구현했다:

요소정의PDA 적용
정보의 비대칭성 관객은 위험을 알지만, 등장인물은 모른다 노이즈에 숨은 적 → 플레이어는 모름, 관전자는 앎
지속적인 기대감 유발 결과를 모르는 불확실성이 지속됨 채널링 4초 동안 "성공할까? 끊길까?" 지속
등장인물에 대한 관심 인물에 감정 이입해야 긴장감 발생 내 해커, 내 팀 → 자연스러운 감정 이입
위협의 시각화 및 암시 직접 보여주지 않아도 위험을 암시 미니맵 적 표시, 핑 시스템, 노이즈 근처 긴장
예측 속 반전 예측 가능한 전개 + 예상치 못한 반전 채널링 99% 직전 적 도착 / 팀파이트 역전
갈등의 점진적 고조 이야기가 진행될수록 긴장 고조 후반 노드 가치 ↑, 디멘션 브레이커 스폰

특히 "정보의 비대칭성"이 핵심이다:

히치콕: "관객은 폭탄이 있다는 걸 안다. 등장인물은 모른다."

PDA: "관전자는 노이즈에 적이 숨어있는 걸 안다. 플레이어는 모른다."

e스포츠 관전 시, 양 팀의 시야가 모두 보이기 때문에 관객은 "저기 숨어있는데...!"라며 조마조마하게 된다. 플레이어들은 모른다. 이것이 노이즈 시스템이 만들어내는 정보 비대칭이다.

\[ \boxed{\text{노이즈 = 정보 비대칭 = 서스펜스의 구조적 기반}} \]

3. 배틀그라운드 분석

3.1 두 가지 긴장감 레이어

레이어 1: 블루존 (서스펜스)

요소 상태
언제 줄어드나? 안다 (타이머)
어디로 줄어드나? 안다 (미니맵)
안 움직이면? 죽는다

“3분 후에 죽음이 온다. 똑딱똑딱…”

\[ \boxed{\text{블루존 = 예정된 죽음의 카운트다운 = 서스펜스}} \]

레이어 2: 적 위치 (불확실성)

상황 정보
능선 너머 모른다
저 집 안 모른다
발소리 주인 모른다

\[ \boxed{\text{적 위치 = 불확실성의 공포}} \]

3.2 배틀그라운드의 천재성

\[ \boxed{\text{서스펜스(안다) + 불확실성(모른다) = 긴장감 레이어링}} \]

“블루존 줄어드니까 움직여야 해” (서스펜스) “근데 움직이면 어디서 총 맞을지 몰라” (불확실성)

두 긴장감이 동시에 작동합니다.

4. LoL 라인전 분석

4.1 “안다” - 타이머 기반 서스펜스

요소 정보
스킬 쿨다운 안다
소환사 주문 (플래시 5분) 안다
드래곤/바론 타이머 안다
정글 몬스터 리스폰 안다
미니언 웨이브 주기 안다
포탑 사거리/공격력 안다
상대 아이템 (탭) 안다

“플래시 30초 후 돌아온다. 그때 이니시 간다.” “드래곤 1분 후 리스폰. 모여야 합니다.” “궁 쿨 돌아오면 한타 가능.”

\[ \boxed{\text{LoL = 타이머 기반 서스펜스}} \]

4.2 “모른다” - 불확실성의 읽기 싸움

상황 정보
상대가 언제 이니시 거나? 모른다
언제 라인 밀고 귀환하나? 모른다
갱킹이 언제 오나? 모른다
정글러 보였는데, 갱? 파밍? 모른다

\[ \boxed{\text{LoL = 위치/의도 불확실성}} \]

4.3 LoL의 긴장감 구조

\[ \boxed{\text{LoL = 타이머 서스펜스 + 위치/의도 불확실성}} \]

배틀그라운드와 마찬가지로, LoL도 두 레이어의 긴장감을 동시에 운용합니다.

5. PDA 구조 분석

5.1 채널링 = 서스펜스 + 불확실성의 결합

“안다” (서스펜스): > “채널링이 성공하면 전장이 뒤집힌다.” > “채널링 시간이 3초라는 걸 안다.” > “게이지가 차오르는 걸 본다.”

“모른다” (불확실성): > “상대 암살자가 노이즈(부쉬)에 대기 중일까?” > “상대가 리스크를 감수하고 들어올까?” > “끊기나? 성공하나?”

요소 히치콕 서스펜스 PDA 채널링
정보 폭탄 있음을 안다 성공하면 뒤집힌다는 걸 안다
불확실성 언제 터지나? 상대가 오나?
시간 경과 똑딱똑딱 게이지 차오름
결과 폭발 or 해제 성공 or 끊김

\[ \boxed{\text{채널링 = "이기면 뒤집힌다(안다)" + "적이 올까?(모른다)"}} \]

채널링 하나에 서스펜스와 불확실성이 동시에 작동합니다.

5.2 CCTV = 불확실성 관리 장치

CCTV 상태 정보 긴장감
장악 적 위치 안다 서스펜스만 (관리 가능)
미장악 적 위치 모른다 서스펜스 + 불확실성 (극대화)

\[ \boxed{\text{CCTV = 불확실성을 제거하여 리스크를 관리하는 장치}} \]

CCTV 없이 채널링? > “성공하면 뒤집힌다” (안다) > “적이 올까?” (모른다) > “노이즈에 숨어있을까?” (모른다) > “몇 명이 올까?” (모른다)

불확실성 폭발 → 긴장감 극대화

5.3 PDA의 긴장감 레이어링

상황 서스펜스 불확실성 긴장감
채널링 + CCTV 있음 ✓ (성공하면 뒤집힌다) △ (적 위치는 안다) 중간
채널링 + CCTV 없음 ✓ (성공하면 뒤집힌다) ✓ (적이 올까? 모른다) 극대화
이동 중 + CCTV 없음 ✗ ✓ (적 위치 모른다) 경계

\[ \boxed{\text{채널링 하나에 서스펜스와 불확실성이 동시 작동}} \]

\[ \boxed{\text{CCTV가 없으면 불확실성까지 추가되어 긴장감 극대화}} \]

6. 세 게임 비교

게임 서스펜스 (안다) 불확실성 (모른다)
배틀그라운드 블루존 적 위치
LoL 쿨다운/오브젝트 타이머 갱킹, 이니시, 의도
PDA 채널링 (성공하면 뒤집힌다) CCTV 미장악 (적이 올까?)

6.1 PDA의 차별점

배그와 LoL은 서스펜스와 불확실성이 별개 시스템에서 온다: - 배그: 블루존(서스펜스) ≠ 적 위치(불확실성) - LoL: 타이머(서스펜스) ≠ 갱킹(불확실성)

PDA의 채널링은 하나의 행동에 둘 다 내장: - “성공하면 뒤집힌다” (서스펜스) - “적이 올까?” (불확실성)

\[ \boxed{\text{채널링 = 서스펜스와 불확실성의 결합체}} \]

6.2 PDA의 추가 요소

영화 용어 PDA 대응 역할
서스펜스 + 불확실성 채널링 긴장감 유발 (결합)
불확실성 관리 CCTV 리스크 조절
미장센 색깔 상황 전달
맥거핀 오브젝트 행동 유발

\[ \boxed{\text{PDA = 서스펜스 + 불확실성 + 미장센 + 맥거핀}} \]

히치콕 영화의 핵심 기법들이 게임 구조에 내장되어 있습니다.

8. 결론

8.1 검증된 공식

배틀그라운드와 LoL이 성공한 이유:

\[ \boxed{\text{서스펜스 + 불확실성 = 긴장감의 레이어링}} \]

8.2 PDA의 진화

PDA는 이 검증된 공식을 하나의 행동(채널링)에 압축했다:

\[ \boxed{\text{채널링 = 서스펜스(성공하면 뒤집힌다) + 불확실성(적이 올까?)}} \]

여기에 히치콕 영화의 기법을 추가했다:

\[ \boxed{\text{미장센(색깔) + 맥거핀(오브젝트)}} \]

8.3 히치콕의 교훈

\[ \boxed{\text{"폭탄을 터뜨리는 게 중요한 게 아닙니다. 폭탄이 있다는 걸 알려주는 게 중요하다."}} \]

PDA의 채널링은 바로 그 “폭탄이 있다는 걸 알려주는” 장치입니다.

8.4 최종 답변

“이 게임이 재미있을까?”

\[ \boxed{\text{서스펜스 구조가 작동하는 한, 재미있습니다.}} \]

본 문서는 게임 재미의 구조적 분석입니다. 실제 재미는 플레이 테스트를 통해 검증되어야 합니다.

자원 경제 시스템 (Resource Economy System)

1. 팀 레벨 시스템

PDA에서 레벨은 개인이 아닌 팀 단위다.

  • 노드 점령, 다운링크 등으로 팀 경험치 획득
  • 팀 전체가 동시에 레벨업
  • 개인이 "더 가져갈" 수 없음

설계 의도:

개인 레벨 시스템을 먼저 설계했으나 폐기했다. 이유:

  1. "캐리" 문제 - 한 명이 레벨 높으면 그 사람만 싸우고 나머지는 구경
  2. 기여 형태 제한 - 다운링크/데미지만 가치 있음

단, 개인 캐리 자체를 막지는 않는다. 솔로 다운링크의 가치는 높게 설계되어 있다. 다만 그 보상을 팀 전체가 동일하게 받는다.

이것은 스노볼 억제가 지나치게 작용하는 것을 방지하는 수단이기도 하다. 개인 플레이어의 압도적 퍼포먼스는 인정하되, 그 결과물은 팀 자원으로 환원된다.

팀 레벨로 전환 시:

  • 서포터가 와딩해도 팀 레벨 상승
  • 탱커가 맞아줘도 팀 레벨 상승
  • 기여의 형태가 다양해진다

2. 무기 레벨 시스템과 매몰비용

2.1 기본 구조

  • 팀 레벨 상승 시 무기 레벨업 포인트 획득
  • 포인트를 현재 장착 무기에 투자
  • 투자한 레벨은 해당 무기에 귀속

2.2 매몰비용(Sunk Cost) 구조

예시 상황 (해커: 별):

Stellar Staff에 레벨 5 투자 (원거리 후방 포지션)
→ 게임 중반, 상대 팀 구성상 Astral Blade(근접 돌입)가 더 유리함을 인식
→ 베이스에서 무기 교체 가능
→ 문제: Stellar Staff 레벨 5는?
→ 사라짐 (또는 대폭 감소)

이것이 매몰비용(Sunk Cost)이다.

설계 의도:

레벨이 유지되면 무기 교체가 공짜다. 공짜면 고민이 없다.

매몰비용이 있으면:

  • "Staff 레벨 5 버리고 Blade 레벨 1로 갈까?"
  • "아니면 Staff로 버틸까?"

이 고민 자체가 게임이다. 정답이 있으면 게임이 아니다. 트레이드오프가 있어야 게임이다.

2.3 매몰비용 오류를 통한 스노볼 억제

인간은 매몰비용을 무시하지 못한다:

팀 상태투자량매몰비용전환 용이성
이기는 팀많음큼어려움
지는 팀적음작음쉬움

결과:

  • 이기는 팀: 매몰비용 함정에 갇힘 → 경직
  • 지는 팀: 자유로운 전환 가능 → 유연

→ 심리를 통한 스노볼 억제 메커니즘

채널링 실패와 방어력의 재정의

저자 주:

해킹 채널링의 중요한 요소 중 하나인 후딜레이(Delay)가 기존 모델에서 빠져 있었다. 최종 검토 중 이를 발견하고 가이드라인 정도는 제시해야 할 것 같아서 정리하던 중, 갑자기 유레카가 터졌다.

"이거, 사실 방어력에 연관된 함수 아닌가?"

기존 모델에서 방어력(\(R\))은 의도적으로 제외되었다 (Q2 참조). 그 결정은 틀리지 않았다—채널링 성공 과정에서는 방어력이 개입하면 메타가 고착된다.

그러나 채널링 실패 후의 상황은 다르다. 후딜레이 동안 보호막은 이미 0이다. 이때 생존 여부를 결정하는 것은 오직 방어력뿐이다.

사고 과정을 남기기 위해 Q&A의 "방어력은 제외하였다"는 삭제하지 않고 그대로 둔다. 구현하시는 분들께 이 발견 과정이 도움이 되길 바란다.

1. 전투 스탯 정의

1.1 확정 변수

기호이름정의단위
\(P\)공격력 (Power)평타 1회당 피해량HP (절대값)
\(S\)보호막 (Shield)채널링 중 흡수 가능량HP
\(I\)Integrity현재 체력 (기준값 100)HP

1.2 미정 변수

기호이름정의비고
\(R\)방어력 (Resistance)피해 감소율% 단위 (0~100)

Note: 방어력(\(R\))은 "피해를 덜 받는다"는 직관적 의미는 있으나, 게임 메커니즘 내에서의 전략적 역할은 아직 명확히 정의되지 않았다. 기존 모델에서는 의도적으로 제외되었다 (Q2 참조).

1.3 피해 계산

방어력 적용 시:

\[\text{실제 피해} = \text{원본 피해} \times (1 - R)\]

예시:

  • \(R = 10\%\): 피해의 90%를 받음
  • \(R = 40\%\): 피해의 60%만 받음

2. 채널링 실패와 후딜레이

2.1 후딜레이 정의

기호이름정의단위
\(t_d\)후딜레이 (Delay)채널링 실패/취소 후 무방비 시간초

2.2 설계 원칙

후딜레이는 게임 템포를 결정하는 핵심 변수 중 하나다.

핵심: 후딜레이는 고정값이어야 한다. 변동하면 유저가 비용을 계산할 수 없다.

개발팀 워크플로우 (상세 내용은 "개발팀을 위한 가이드" 참조):

  • Top-down: 게임 템포 설계 시 \(T^*\), \(t_d\)를 먼저 결정하고 나머지를 역산
  • Bottom-up: 개별 챔피언 추가 시 \(v\), \(r\)을 먼저 결정하고 순차 도출 (\(T^*\)는 이미 결정됨)

2.3 후딜레이 중 상태

채널링이 중단되면:

  • 보호막(\(S\)): 이미 0 (소진 또는 무효)
  • Integrity(\(I\)): 현재값 유지
  • 행동: 불가 (무방비 상태)

이 시간 동안 적의 집중 화력에 노출된다.

2.4 예상 피해량

후딜레이 동안 받는 피해:

\[D_{delay} = t_d \times \overline{DPS}\]

여기서 \(\overline{DPS}\)는 적의 평균 초당 화력이다.

잠깐.

이 상황에서 보호막(\(S\))은 이미 0이다. Integrity(\(I\))만 남았다.

그렇다면 방어력(\(R\))은?

3. 방어력의 재발견: 실패 비용 함수

3.1 발견

Section 7에서 "역할 미정"으로 남겨뒀던 방어력(\(R\))의 의미가 여기서 드러난다.

채널링 실패 시:

  • 보호막(\(S\)) = 0 (이미 소진)
  • 행동 불가 (후딜레이 \(t_d\) 동안)
  • \(R\)이 유일한 피해 감소 수단

3.2 실패 비용 함수

\[D_{fail} = t_d \times \overline{DPS} \times (1 - R)\]

다운 확률:

\[P_{down} = \min\left(1, \frac{D_{fail}}{I}\right) = \min\left(1, \frac{t_d \times \overline{DPS} \times (1-R)}{I}\right)\]

Note: \(P_{down}\)은 실제 "확률"이 아니라 위험도 지표(Risk Index)다. 결정론적 게임에서 \(D_{fail} < I\)이면 다운되지 않고 체력만 감소하지만, 이 값이 1에 가까울수록 "실패 시 치명적"임을 의미한다.

3.3 역할의 분리

3.4 역할별 플레이 스타일

A. 탱커 (San) - "보험 빵빵함"

  • 특징: \(R\) 높음 (40%+), \(S\) 높음, \(v\) 낮음
  • 수식: \(D_{fail}\) 값이 작음
  • 스타일: "실패해도 안 죽어. 과감하게 들어갈게."

B. 암살자 (Bam) - "보험 없음"

  • 특징: \(R\) 낮음 (10%), \(S\) 낮음, \(v\) 높음
  • 수식: \(D_{fail}\) 값이 매우 큼 (맞으면 다운)
  • 스타일: "끊기면 끝장. 무조건 성공시키거나, 안 들켜야 해."

        S (보호막)        R (방어력)
             ↘             ↙
           [양의 상관관계]
            (결과적으로)

핵심 관계:

  • \(v\)와 \(S\): 역비례 (k_champ 공식: \(k = \bar{v}/v_{new}\))
  • \(v\)와 \(R\): 역비례 (역할 설계상)
  • 결과적으로 \(S\)와 \(R\): 양의 상관관계를 지향

주의: 이것은 하드코딩된 공식이 아니라 밸런싱 가이드라인이다. \(S\)와 \(R\)이 직접 연결되는 것이 아니라, \(v\)를 매개로 한 간접 관계다.

3.6 설계 근거: 왜 탱커는 보호막도 높은가?

표면적으로는 "방어력이 높으니까 보호막은 낮아도 되지 않나?"라고 생각할 수 있다.

그러나:

  1. 탱커는 이동속도(\(v\))가 낮다.
  2. 이동속도가 낮으면 해킹 시도 빈도(Frequency)가 적다.
  3. 어렵게 잡은 기회이므로, 보호막을 높여서 성공 확률을 보장해야 한다.

만약 탱커의 보호막이 낮다면?

  • 느려서 잘 못 감
  • 도착해서 해킹 눌렀는데 평타 몇 대 맞고 끊김
  • 방어력 높아서 안 죽음
  • 결론: 안 죽는데 해킹도 못 하는 샌드백

탱커가 위협적이려면:

"쟤는 느려서 잘 안 오지만, 일단 붙어서 해킹을 시작하면 떼어내기가(보호막 까기가) 정말 힘들다"

이 공포가 있어야 한다.

시스템 상태의 정량화: 등가 DPS 모델

저자 주:

테크니컬 문서를 마무리한 후, 한 가지 빠진 요소가 있음을 깨달았다. 시스템 상태(CC)다.

Lock, Drag, Jam, Standby 같은 행동 방해 스킬들은 어떻게 방정식에 편입시킬 것인가? 고민하던 중, 핵심을 깨달았다.

"하드 CC는 채널링을 즉시 취소시키고 보호막을 0으로 만든다. 이건 사실상 '남은 보호막 전체를 즉시 깎는 것'과 동일하다."

1. 시스템 상태(CC)의 정의

PDA에서 시스템 상태는 두 가지 범주로 나뉜다:

1.1 하드 CC (채널링 즉시 취소)

상태효과채널링예시
Lock (경직)모든 행동 불가즉시 취소, S→0락의 Direct Hit
Standby (수면)피격 전까지 행동 불가즉시 취소, S→0소리의 Lullaby

1.2 소프트 CC (채널링 유지)

상태효과채널링예시
Drag (둔화)이동속도 감소유지바다의 Hydro Field
Jam (방해)스킬 사용 불가유지 (평타는 가능)락의 Sound Wave

핵심 차이: 하드 CC는 채널링을 즉시 취소하고 보호막을 0으로 만든다. 소프트 CC는 행동만 제한하고 채널링은 유지된다.

2. 등가 DPS 공식

2.1 하드 CC의 등가 가치

하드 CC가 채널링 중인 적에게 적중하면:

  • 채널링 즉시 취소
  • 보호막 \(S\) → 0
  • 남은 보호막이 얼마든, 전부 무효화
  • + 지속시간 동안 행동 불가

따라서 하드 CC 1회의 등가 가치는 두 효과의 합이다:

\[\text{Value}_{hard} = \underbrace{\frac{S}{2}}_{\text{보호막 무효화}} + \underbrace{t_{cc} \times \overline{DPS}_{team} \times k_{effect}}_{\text{지속시간 동안 추가 피해}}\]

평균적으로 보호막이 절반 남았을 때 적중한다고 가정하면, 하드 CC 1회 = \(S/2\) + 지속시간 효과와 등가다.

왜 이렇게 복잡한 제약이 붙는가?

  • 소프트 CC: 지속시간 효과만 → 단일 항
  • 하드 CC: 보호막 무효화 + 지속시간 효과 → 두 효과를 동시에 가짐

즉, 하드 CC는 "두 가지 스킬을 한 번에 쓰는 것"과 같다. 그래서 제약도 두 배로 걸어야 공평하다.

2.2 소프트 CC의 등가 가치

소프트 CC는 채널링을 취소하지 않으므로, 그 시간 동안 팀이 추가로 가할 수 있는 피해로 환산한다:

\[\text{Value}_{soft} = t_{cc} \times \overline{DPS}_{team} \times k_{effect}\]

기호의미
\(t_{cc}\)상태 지속 시간 (초)
\(\overline{DPS}_{team}\)팀 평균 DPS (Integrity/초)
\(k_{effect}\)효과 강도 계수 (0~1)

2.3 효과 강도 계수

(예시 수치 — 실제 값은 플레이 테스트로 결정)

  • Drag 30% → \(k \approx 0.3\)
  • Drag 50% → \(k \approx 0.5\)
  • Jam (스킬 봉인) → \(k \approx 0.3\)

3. 밸런싱 적용

3.1 예시 계산

주의: 아래 모든 수치는 예시일 뿐입니다. 실제 값은 플레이 테스트로 결정되어야 합니다.

가정:

  • 기준 보호막 \(S = 50\) Integrity
  • 팀 평균 DPS = 10 Integrity/초

\[\text{Value} = \frac{S}{2} + t_{cc} \times \overline{DPS} \times k = \frac{50}{2} + 1.5 \times 10 \times 1.0 = 25 + 15 = 40 \text{ Integrity}\]

바다의 Drag (소프트 CC, 2초, 50%):

\[\text{Value} = 2 \times 10 \times 0.5 = 10 \text{ Integrity}\]

결론: 하드 CC 1회는 소프트 CC 여러 회에 해당하는 가치를 가진다. 따라서 하드 CC는 긴 쿨다운으로 제약해야 한다.

1. 핵심 철학

휴먼 에러는 제거 대상이 아니라, 시스템 변수다.

밸런스 패치가 아니라, 매칭으로 해결한다.

2. 변수 정의

변수이름정의
\(P\)퍼포먼스 점수해당 판의 종합 기여도
\(\bar{P}\)기대 퍼포먼스해당 MMR 구간의 평균 \(P\)
\(\sigma\)일관성 지표최근 N판 \(P\)의 표준편차
\(\bar{\sigma}\)티어 평균 분산해당 티어의 평균 \(\sigma\)

3. 퍼포먼스 점수 \(P\)

\[ P = \sum_i w_i \cdot x_i \]

지표 (\(x_i\))로그 출처
해킹 성공률해킹 시작/취소/성공
팀 기여도점유, 브레이커, 가디언 참여

팀 동조율핑 발생/수락/무시

가중치 \(w_i\)는 역할별로 다르게 설정 가능.

4. 두 개의 가속 항

4.1 괴리도 \(\Phi\)

\[ \Phi = \frac{|P - \bar{P}|}{\bar{P}} \]

현재 위치가 얼마나 잘못됐는가

상황\(\Phi\)의미
스머프↑↑기대보다 훨씬 잘함
트롤/부스팅↑↑기대보다 훨씬 못함
적정 위치≈ 0기대만큼 함

4.2 불안정도 \(\Sigma\)

\[ \Sigma = \frac{\sigma}{\bar{\sigma}} \]

얼마나 들쭉날쭉한가

상황\(\Sigma\)의미
컨디션 기복> 1티어 평균보다 불안정
일관된 플레이< 1티어 평균보다 안정적
평균= 1티어 기준

5. MMR 가속도 공식

\[ \Delta MMR_{final} = \Delta MMR_{base} \times (1 + \lambda_1 \cdot \Phi + \lambda_2 \cdot \Sigma) \]

항역할
\(\Delta MMR_{base}\)승패 기본 변동 (예: ±20)
\(\lambda_1 \cdot \Phi\)괴리도 가속 (스머프/트롤 감지)
\(\lambda_2 \cdot \Sigma\)불안정도 가속 (빠른 수렴)

6. 작동 예시

플레이어\(\Phi\)\(\Sigma\)결과

7. 수렴 메커니즘

적정 위치에 도달하면:

  1. \(P \approx \bar{P}\) → \(\Phi \to 0\)
  2. \(\sigma \approx \bar{\sigma}\) → \(\Sigma \to 1\)
  3. \(\Delta MMR_{final} \to \Delta MMR_{base}\) (가속도 소멸)

결과: 자동 안착

8. 매칭 적용

\(\sigma\)를 매칭에도 사용:

\(\sigma\) 비슷한 플레이어끼리 매칭

\(\sigma\) 낮음 vs \(\sigma\) 낮음 = 정석 대결

\(\sigma\) 높음 vs \(\sigma\) 높음 = 혼돈의 카오스

플레이 스타일 매칭

9. 요약

\[ \boxed{\text{휴먼 에러 = 밸런스 변수 ❌ \quad 휴먼 에러 = 매칭 변수 ⭕}} \]

  • \(\Phi\) (괴리도) → 잘못된 위치 감지 → 빠른 이동
  • \(\Sigma\) (불안정도) → 수렴 안 됨 감지 → 빠른 탐색

예상 질문 & 답변 (Q&A)

본 문서의 수학적 모델링에 대해 제기될 수 있는 학술적 질문들과, 그에 대한 설계 의도를 정리합니다.

Q1. 반응-확산 방정식에서 정상 상태(Steady State)를 가정한 것은 물리적으로 엄밀하지 않은 것 아닌가요?

A: 의도적인 설계입니다.

만약 과도 응답(Transient Response)을 서버에서 실시간으로 계산하려면, 매 틱(Tick)마다 수치 해석(Numerical Analysis)을 돌려야 합니다. 8명의 플레이어와 여러 노드를 처리하면 CPU 부하가 폭발합니다.

정상 상태를 가정함으로써 복잡한 미분방정식을 단순한 대수 방정식 \(T^* = \alpha t + \gamma w\)으로 치환했습니다. 서버는 \(O(1)\)의 비용으로 즉시 채널링 시간을 계산할 수 있습니다.

또한, 채널링 시간은 변칙적인 값이 아니라 맵 구조, 챔피언 기본 변수, 노드의 가치와 위치에 의해 미리 정해지는 '상수값'이어야 합니다. 플레이어가 예측 가능한 전략을 세우려면 채널링 시간이 고정되어 있어야 하기 때문입니다. 이 설계 철학이 정상 상태 가정을 더욱 정당화합니다.

결론: 이것은 '수학적 비약'이 아니라 '공학적 압축(Engineering Optimization)'입니다.

Q2. 란체스터 제곱 법칙을 사용하면서, 채널러 보호막은 선형 DPS로 계산하는 것이 모순 아닌가요? 또한 방어력은 왜 제외했나요?

A: 모순이 아니라 의도적인 분리입니다.

  • 전선 교전 ((A-1) vs B): 서로 공격하므로 → 제곱 법칙
  • 채널러 공방: 일방적으로 맞기만 하므로 → 선형 법칙 (DPS 누적)

방어력을 제외한 것도 의도적입니다. 방어력이 보호막(S)에 반영되면, 탱커만 해킹을 담당하게 됩니다(Meta Fixation). 모든 포지션이 상황에 따라 해커가 될 수 있는 '전략적 유동성'을 위해 방어력 변수를 소거했습니다.

결론: 수학적 단순화가 아니라, 메타 고착화 방지를 위한 게임 디자인적 결단입니다.

Q3. 스노볼 모델에서 \(\mu\)가 레벨에 종속되어야 수학적으로 완벽한 포화(Saturation)가 되는 것 아닌가요?

A: 수학적으로는 맞지만, 의도적으로 선형 모델을 택했습니다.

비선형 모델에서는 격차가 커질수록 변화율 \(dG/dt\)이 폭발적으로 커집니다. 이 경우 "게임이 교착 상태인지"를 서버가 판단하기 어렵습니다(노이즈가 심함).

선형 모델에서는 기울기가 0에 수렴하는 순간 \(dG/dt \approx 0\)이 명확하게 드러납니다. 이래야 서버가 "지금 게임이 루즈하다"는 신호를 오탐(False Positive) 없이 감지하고, 브레이커(Dimension Breaker)를 정확한 타이밍에 투입할 수 있습니다.

참고로, 폭발(발산)은 최종적으로 \(L_{max}\)에 의해 제한되지만, \(L_{max}\) 도달 이전 구간에서는 \(\mu\) 및 플레이어 실력 편차로 인해 기울기 변동(노이즈)이 커질 수 있습니다. 따라서 여기서 문제 삼는 건 '무한 발산'이 아니라 교착 감지 신호의 안정성(SNR)입니다.

실제로 본문에서도 "유리한 팀이 다음 자원을 먹기 쉽다"고 적었습니다. 실제 곡선은 상한선이 존재하는 지수함수꼴에 가까울 것입니다. 그러나 계산 자원을 아끼고 교착 감지의 정확도를 높이기 위해 선형이라는 가정을 채택했습니다.

결론: 현실성(Accuracy)보다 '게임 템포 제어 가능성(Controllability)'을 우선한 설계입니다.

Q4. 실제 게임 데이터는 불연속인데 \(dG/dt\)를 어떻게 계산하나요?

A: 미분이 아니라 평균변화율(\(\Delta G / \Delta t\))로 계산합니다.

물론 학술적으로는 \(G(t)\)를 헤비사이드 계단 함수(Heaviside Step Function)의 합으로 보고, 이를 푸리에 변환(Fourier Transform)하여 주파수 도메인에서 분석하는 방법도 있습니다. 하지만 실시간 게임 서버 환경에서는:

  1. 계산 비용(Computational Cost): \(O(N \log N)\)의 FFT 연산 부하
  2. SNR(Signal-to-Noise Ratio): 델타 함수로 인한 고주파 노이즈 문제

위 두 이유로 실시간 환경에서는 과잉 설계입니다. 따라서 가장 가볍고 직관적인 시간 도메인의 평균변화율(Time-domain Average Rate) 방식을 채택했습니다.

실제 \(G(t)\)는 선형 + 불연속 함수입니다. 킬, 노드 점령 등 이벤트마다 계단식으로 점프합니다. 본문에서는 그래디언트 기반으로 서술했지만, 실제 구현에서는 시간 윈도우 내 평균변화율만 측정해도 충분합니다. 선형 모델이기 때문에 평균이 대표값으로서 의미를 갖습니다.

다만 엣지 케이스가 존재합니다:

60초 윈도우 내 평균변화율 ≈ 0인데, 실제로는 30초 부근에 대규모 교전이 발생해 \(G(t)\)가 급등 후 원래 값으로 복귀한 경우

이 경우 "교착"이 아니라 "격렬한 공방"입니다. 이를 구분하기 위해 듀얼 윈도우(Dual Window) 방식을 제안합니다:

윈도우범위역할
Window A0~30초최근 추세
Window B30~60초이전 추세

두 윈도우 모두 \(\Delta G / \Delta t \approx 0\)일 때만 교착으로 판정합니다.

또한 감지 즉시 스폰하지 않습니다:

교착 감지 → "브레이커 스폰까지 15초" 알림 → 스폰

설계 의도:

  • 플레이어에게 준비 시간 제공
  • 양 팀 모두 동등한 정보 → 공정성
  • "곧 판이 바뀐다"는 예고 → 서스펜스

결론: 선형화의 진짜 이점은 플레이테스트 몇 판만으로 임계값 설정이 가능하다는 점입니다. 비선형이었다면 윈도우 크기와 임계값 튜닝이 상황 의존적이 되어 복잡해졌을 것입니다.

위 Q&A는 학술적 검토 과정에서 제기될 수 있는 예상 질문들입니다. 모든 '비약'은 서버 퍼포먼스와 게임 경험을 위한 의도된 설계임을 명시합니다.

개발팀을 위한 가이드

이 문서를 실제로 구현하실 엔지니어분들께:

핵심은 "두 가지 워크플로우"입니다.

A. 게임 템포 설계 시 (Top-down)

"한 판 15분, 채널링 5초, 교전 10초를 원한다" — 게임 전체의 리듬을 먼저 잡을 때.

  1. 역할별 총 기여도를 동일하게 설정한다 (공리)
  2. 원하는 게임 템포에서 \(T^*\), \(t_d\) 결정
  3. \(T^*\)에서 평균 \(\bar{v}\), \(\bar{r}\), \(\overline{DPS}\), \(\bar{S}\) 역산
  4. CC 예산 배분 (하드 CC 보유 시 DPS 낮게)

\[ T^*, t_d \xrightarrow{\text{역산}} \bar{v}, \bar{r}, \overline{DPS}, \bar{S}, \text{CC 예산}, ... \]

B. 개별 챔피언 추가 시 (Bottom-up)

"이 챔피언은 빠른 암살자야, v = 450, r = 300" — 개별 챔피언을 추가할 때.

B-1. 기존 방식 (v0.5)

  1. 역할별 총 기여도를 동일하게 설정한다 (공리)
  2. \(v\), \(r\) 결정 (이 챔피언의 컨셉)
  3. \(v, r \rightarrow k_{champ} \rightarrow S \rightarrow DPS\) 순차 도출
  4. CC 예산 배분 (하드 CC ↔ DPS 트레이드오프)
  5. \(T^*\)는 이미 전체 평균에서 결정됨 (개별 챔피언이 바꾸지 않음)

\[ v, r \xrightarrow{\text{순차 도출}} k_{champ} \rightarrow S_{champ} \rightarrow DPS_{required} \]

B-2. 능력치 버젯 모델 (v0.6)

6개 스탯 + CC를 하나의 버젯으로 통합.

버젯 공식: (단위 = 총 데미지)

\[ Budget = \underbrace{(ATK \times AS + CC_{equiv})}_{\text{공격력}} \times \underbrace{\left( \frac{d-r}{v} + \frac{S}{DPS_{avg}} \right)}_{\text{공격 시간}} \times \underbrace{\frac{I}{t_d \times DPS_{avg} \times (1-R)}}_{\text{실패 허용 횟수}} \]

항의미
\(ATK \times AS + CC_{equiv}\)총 공격력 (순수 DPS + CC 등가)
\(\frac{d-r}{v} + \frac{S}{DPS_{avg}}\)공격 시간 (도착 + 채널링)
\(\frac{I}{t_d \times DPS_{avg} \times (1-R)}\)실패 허용 횟수

CC 등가: \( CC_{equiv} = \frac{S/2 + t_{cc} \times DPS_{avg} \times k}{t_{cooldown}} \)

사용법:

  1. Budget 값을 먼저 고정 (모든 챔피언 동일)
  2. 7개 스탯 (ATK, AS, v, r, S, R, CC) 자유롭게 분배
  3. Budget만 맞으면 밸런스 보장

고정값: \(d = 105\), \(I = 100\), \(t_d = 1.5\)초

요약

상황워크플로우시작점
게임 전체 템포 설계Top-down\(T^*\), \(t_d\)
챔피언 하나 추가Bottom-up (Budget)\(Budget\) → 7개 스탯 분배

두 방법 모두 유효하며, 같은 균형 상태로 수렴합니다.

정리

  • Top-down: 전체 템포를 잡거나 수정할 때, 채널링 시간 \(T^*\)가 평균 이동속도 \(\bar{v}\), 평균 사거리 \(\bar{r}\)을 포함한 모든 파라미터를 연쇄적으로 결정합니다.
  • Bottom-up (Budget): 개별 해커를 설계할 때, Budget을 먼저 고정하고 7개 스탯 (ATK, AS, v, r, S, R, CC)을 자유롭게 분배합니다. Budget = 공격력 × 공격시간 × 실패허용횟수만 맞으면 무차별곡선 위의 어떤 조합이든 밸런스가 보장됩니다.

방어력(\(R\))은 "채널링 실패와 방어력의 재정의" 섹션에서 다루었듯이 "채널링 실패 시 생존 스탯"으로 정의됩니다. 이동속도(\(v\))와 역비례하도록 설계하면 역할별 차별화가 자연스럽게 이루어집니다.

하드 CC 예산 배분

하드 CC는 "채널링을 즉시 끊는다"는 점에서 극도로 강력합니다.

  • 아군 채널링 보호: 적 해커에게 Lock → 채널링 취소 → 아군 채널링 성공
  • 적 채널링 방해: 적 채널러에게 Lock → 보호막 무효화 → 해킹 실패

따라서 하드 CC 보유 해커는:

  1. 긴 쿨다운으로 희소성 부여
  2. 낮은 순수 DPS로 트레이드오프
  3. "언제 쓸 것인가"가 핵심 의사결정이 되도록

설계 원칙: 하드 CC를 가진 캐릭터의 공격력을 올려주고 싶다면 방법은 하나뿐이다. CC기의 쿨타임을 늘려서 등가 DPS를 깎는 수밖에 없다. DPS를 올리면서 CC도 유지하는 건 예산 초과다.

예시) 하늘: 하늘의 궁극기는 매우 강력하다. 글로벌 궁극기로, 타겟에게 하드 CC를 걸어 채널링을 취소시킨다. 하지만 하늘은 원거리 딜러이므로 자체 DPS는 유지되어야 한다. 이걸 유지하는 방법은 하나뿐이다. 궁극기의 쿨타임을 극단적으로 늘려서 궁극기를 제외한 DPS를 유지시켜야 한다.

핵심: 이론적으로 하드 CC는 \(S/2 + t_{cc} \times \overline{DPS} \times k\)의 등가 가치를 가지며, 이는 순수 DPS 수 초 분량에 해당합니다. 하드 CC를 보유한 해커는 그만큼 DPS 예산에서 비용을 지불합니다. 다만 실제로는 더 많은 변수가 개입할 수 있으므로, 최종 수치는 개발팀 여러분의 판단을 신뢰합니다.

구현 방법 제안

위 워크플로우에서 도출된 수치들을 실제 서버/클라이언트에 어떻게 올릴 것인가?

왜 이렇게 해야 하는가

  • 보안: 방정식을 클라이언트에 직접 입력하면 역공학(Reverse Engineering)을 통한 밸런스 수치 노출 위험이 있습니다.
  • 성능: 서버에서 매번 행렬 연산을 돌리는 것은 불필요한 부하입니다.
  • 정당성: 채널링 시간, 보호막 계수, DPS 등은 맵/노드/챔피언 구조가 바뀌지 않는 한 상수입니다. 게임 중에 동적으로 변할 이유가 없습니다.

제안: 사전 계산 + 하드코딩

별도의 보안 환경에서 미리 계산하여 테이블을 생성하고, 클라이언트/서버에는 결과값만 하드코딩합니다.

1. 채널링 시간 \(T^*\)

# 사전 계산 (Python/NumPy 의사코드)
A = D_matrix + V_matrix + K_matrix
b = source_vector
U_steady = np.linalg.solve(A, -b)
T_star = extract_characteristic_time(A)

# 런타임 (O(1))
channeling_time = T_star_lookup[node_id]

2. 캐릭터 스탯 예산

본 문서의 밸런싱 체인 \(v \rightarrow k_{champ} \rightarrow S_{champ} \rightarrow DPS_{required}\)도 같은 방식으로 처리합니다.

# 사전 계산 (Python 의사코드)
for champ in champion_list:
    k_champ = v_mean / champ.move_speed
    S_champ = S_mean * k_champ
    DPS_required = S_mean / T_star  # 평균 DPS 기준
    
    # 하드 CC 보유 시 DPS 차감
    if champ.has_hard_cc:
        cc_value = (S_mean / 2) + (champ.cc_duration * DPS_team * k_effect)
        DPS_required -= cc_value / cc_cooldown
    
    balance_table[champ.id] = {
        'k': k_champ,
        'S': S_champ,
        'DPS': DPS_required
    }

# 런타임 (O(1))
shield = balance_table[champ_id]['S']

3. 교착 감지 (그래디언트 스폰)

교착 감지 알고리즘(Q4 참조)은 위와 다릅니다. 클라이언트에 하드코딩하지 않고, 서버에서 실시간으로 계산합니다.

  • 이유: 중요한 알고리즘이 클라이언트에 노출되면 역공학 위험이 있습니다.
  • 원칙: 클라이언트는 멍청하게, 서버는 똑똑하게.
  • 구현: 서버가 교착 판정 → 클라이언트에 스폰 패킷 전송 → 클라이언트는 연출만 담당.

요약: 방정식으로 도출 → 테이블 생성 → 하드코딩. 이 파이프라인은 채널링 시간, 보호막 계수, DPS, CC 예산 등 본 문서의 모든 밸런스 수치에 동일하게 적용됩니다. 단, 교착 감지처럼 실시간 판정이 필요한 로직은 서버 사이드에서 처리합니다.

실제 템포는 플레이 테스트로 결정되어야 합니다.

그리고 이 방정식은 완전하지 않습니다.

  • 실제 맵은 2차원인데, 본 모델에서는 1차원으로 압축되었습니다. — 단, 제 직관상, 게임에서 중요한 것은 노드 간 최단 경로이므로, 2차원 확장 없이 그래프 기반 접근으로 충분할 수 있습니다.
  • 캐릭터의 다른 스킬 계수나 스탯은 어떻게 방정식에 자연스럽게 편입시킬 것인지 정의되지 않았습니다. → v0.2: 시스템 상태(Lock, Drag, Jam, Standby)의 등가 DPS 모델이 추가되었습니다.

1. 캐릭터 설계 철학

1.0 디자인 가이드

세계관 컨셉:

"현실의 피규어 박스를 뜯어서(Unboxing), 가상의 전장(Plastic Dimension)에 올려놓으면, 그때부터 영혼(Hologram)이 깃들어 움직입니다."

아트 스타일 제안:

아트팀과 그래픽팀의 결정을 전적으로 신뢰하지만, 제안드리고 싶은 방향이 있습니다:

바비인형 느낌 + 카툰 렌더링

플라스틱 피규어 + 홀로그램 무기/이펙트

핵심: 팔아야 한다

픽창 인터페이스:

해커가 나열되는 게 아니라 박스가 나열됩니다.

    1.3 무기 시스템 철학

    무기 선택 = 강약이 아닌 축(Axis)의 선택

    모든 해커는 2가지 무기를 가지며, 이는 "어느 쪽이 강한가"가 아니라 "어떤 방식으로 싸울 것인가"의 선택이다.

    유형무기 A무기 B
    거리원거리근거리
    속도느림 + 강함빠름 + 약함
    역할한 방지속 압박
    범위단일광역
    무기 교체 원칙:

    스탯이 변하고, 스킬이 약간 변하지만, 스킬의 핵심 로직은 변하면 안 된다

    변함변하지 않음
    스탯 (공격력, 사거리 등)스킬의 핵심 로직
    스킬의 형태/범위/속도스킬이 하는 일
    플레이 스타일캐릭터 정체성
    예시 (미르):
  • 잠자는 화산(Magma Drake): 짧고 넓은 브레스
  • 자유로운 벼락(Lightning Wyrm): 얇고 긴 브레스
  • 공통: "전방에 에너지를 뿜는다" (핵심 로직 동일)

    • 1.4 무기 2개 시스템의 전략적 의미

    밴픽 감옥(Draft Prison) 탈출

    기존 MOBA의 고통: 선픽에서 상성이 불리한 캐릭터를 뽑으면 20분 동안 고통받아야 한다.

    PDA의 해법: 귀환/리스폰 시 무기 교체 가능.

    상황대응
    상대 암살자가 나(원거리)를 노린다근거리 무기로 바꿔서 역카운터
    상대가 방패를 들고 버틴다관통형 무기로 교체
    핵심: 절대적인 상성이 사라진다. 불리하면 무기를 바꿔서 판을 엎을 수 있습니다.

    불완전 정보 게임:

    상대가 귀환해서 뭘 들고 나올지 모른다. Tab 키를 눌러도 "현재 무기"만 보이지, "다음 무기"는 모른다.

    내시균형이 존재하겠지만, 인간이 계산 불가능한 영역에 있다

    이 불확실성이 게임을 "고인물도 영원히 연구해야 하는 게임"으로 만든다.

    1.5 레벨 시스템

    팀 레벨 5 + 무기 레벨 5 = 스킬 레벨 10

    구분범위역할
    팀 레벨1~5안전장치 (Base) - 팀이 성장하면 자동 보장
    무기 레벨1~5개인 투자 (Edge) - 내가 직접 투자한 XP
    스킬 레벨2~10총합 - 레벨 2부터 시작
    궁극기 해금:

    본편에서 이미 설명한 내용이지만, 팀 레벨이 2가 되었을 때 궁극기가 사용 가능해진다.

    조건설명
    해금 시점팀 레벨 2
    초기 쿨다운약간 존재 (기습 궁극기 사용 방지)
    레벨 2가 먼저 찍히면 어쨌든 쿨이 먼저 돈다. 먼저 궁극기를 쓸 권한 자체는 주어집니다.

    왜 이 구조인가:

    문제해결
    무기 바꾸면 너무 약해진다팀 레벨 5는 보장 → 절반의 전투력 유지
    바꾸는 게 공짜다무기 레벨 5는 날아감 → 매몰비용 발생
    팀빨 vs 개인기 논쟁5:5로 균형
    직관성: [Lv.7] vs [Lv.9] — 숫자 하나로 전투력 비교 가능.

    1.6 XP 투자 심리전

    먼저 패를 까는 자가 주도권을 잡지만, 카운터 맞을 리스크도 짊어진다

    전략장점단점
    선투자 (Aggressive)초반 주도권 장악정보 노출, 카운터 당함
    후투자 (Reactive)상대 보고 역카운터초반 눌림
    XP 비축 (Hoarding)막판 승부수기본 스탯으로 버텨야 함
    매몰비용 딜레마:

    무기 A에 XP를 투자했는데, 상대가 상성이 유리한 무기로 나왔다.

  • 바꾸면? → 투자한 XP 손해
  • 안 바꾸면? → 상성으로 손해

    • 이 고민 자체가 컨텐츠다

  • 리스크: 길 위의 모든 유닛(아군/적군)은 이동속도가 대폭 증가.
  • "나를 잡으러 와봐" (도발)
  • "따라와, 길 열어줄게" (아군 지원)

    • Q 일반 스킬: 무대(Stage)

    "자, 주목."

    무기스킬명효과
    은반악셀(Axel)"무대: 악셀." 전방으로 우아하게 도약 회전. 착지 지점에 넓은 얼음판(무대) 생성 + 광역 대미지.
    아스팔트드리프트(Drift)"무대: 드리프트." 고속 주행 중 몸을 비틀며 급제동. 바닥을 갈아버리며 부채꼴 범위 마찰열 대미지 + 넉백.
    W 궁극기: 피날레(Finale)

    "이게 내 엔딩이야."

  • 공통: 제자리에서 스핀하며 주변에 지속 대미지. 패시브(길)의 지속시간과 범위가 2배로 증가.

    • 무기스킬명효과
    은반갈라쇼(Platinum Gala)우아하게 스핀. 주변 지속 대미지 + 빙판 생성.
    아스팔트스트릿 잼(Street Jam)격렬하게 스핀. 주변 지속 대미지 + 적 밀쳐냄.

    3. 해커 시너지 예시

    조합시너지
    산 + 호전면 붕괴 + 고정
    락 + 밤혼란 + 고립
    바다 + 하늘구도 설계 + 정보 누적
    별 + 미르리듬 폭주 + 글로벌 압박

    4. 요약

    이름: 한국어 + 영문 표기 용이 + 의미 내포

    무기: 강약이 아닌 축(Axis)의 선택

    스킬: 모든 선택에는 대가가 있다

    5. 밸런싱 고찰

    5.1 강력한 캐릭터의 제어

    강력함 = 긴 쿨다운으로 서스펜스 유지

    미르와 바다는 누가 봐도 강력한 캐릭터다. 글로벌 이동, 전장 흐름 제어 — 이 능력들이 자유롭게 사용된다면 게임의 긴장감은 사라진다.

    5.2 미르의 제약

    스킬제약이유
    W (마실/Sky Glide)쿨다운 매우 김글로벌 이동은 게임을 뒤집는 능력
    P (역린/Reverse Scale)내부 쿨다운 존재피격마다 발동되면 너무 강함
    미르가 하늘을 날 수 있다는 건 강력합니다.
    하지만 "언제 날 수 있는가"가 제한되어야 서스펜스가 생깁니다.

    "미르 궁 있어?" — 이 질문이 의미 있어야 합니다.

    5.3 바다의 역설

    스킬특성
    Q (물바다/Hydro Field)쿨다운 김
    W (비의 행진/Rain Parade)쿨다운 김
    P (물결타기/Water Glide)스킬 사용 시에만 발동
    역설:
  • 바다의 패시브는 "스킬을 써야" 발동됩니다.
  • 하지만 바다의 스킬은 쿨다운이 길다.
  • 따라서 바다는 스킬을 아껴야 하지만, 스킬을 써야 움직인다.

    • 스킬 아끼면 느려지고, 스킬 쓰면 쿨다운에 묶인다

    이 긴장감이 바다를 "강하지만 어려운" 캐릭터로 만든다.

    5.4 설계 원칙

    캐릭터강점제약
    미르글로벌 이동긴 궁/패시브 쿨다운
    바다전장 흐름 제어긴 스킬 쿨다운 + 패시브 역설
    밤암살공격하면 스택 초기화
    산탱킹맞지 않으면 스택 감소
    별폭주리듬 끊기면 약해짐
    락CC박자 놓치면 느려짐
    모든 강력함에는 구조적 제약이 붙는다

    저자 주: 숫자(대미지, 쿨다운 초)는 플레이테스트에서 정해집니다.
    하지만 "왜 이 캐릭터가 이런 제약을 가져야 하는가"는 설계 단계에서 정해져야 합니다.
    숫자는 바뀔 수 있지만, 철학은 바뀌면 안 됩니다.

    부록: 한국어 네이밍 비하인드

    저자 주: 영문 스킬명을 한국어로 옮기는 과정에서의 사고 흐름을 기록합니다.
    디자이너분들께 네이밍 접근법의 참고가 되길 바랍니다.

    네이밍 원칙

    원칙설명예시
    직관성듣자마자 이미지가 그려져야 함산사태, 급강하, 물바다
    콜아웃e스포츠 캐스터가 외치기 좋아야 함"지평선 너머 쏩니다!!!"
    컨셉 연결캐릭터 이름/테마와 연결바다 → 물바다, 산 → 산사태
    톤 통일한 캐릭터 내 어휘 톤 맞추기밤: 비수/암기/잠행/잠적 (전부 한자어)

    케이스 스터디: "지평선 너머" (하늘 W)

    원문: Skyline Lock
    효과: 글로벌 관통 저격 + 위치 노출 + Lock CC

    1차 시도 (한자어 조합):

  • 창공쇄(蒼空鎖) → "읽기 어렵고 직관적이지 않음" → 패스
  • 마천안(摩天眼) → "마천루 + 눈, 근데 뭔 소린지 모르겠음" → 패스

    • 2차 시도 (의미 분해):

  • Skyline = 스카이라인 = 건물 윤곽선? → 아니, 하늘과 땅이 만나는 선
  • Lock = 잠근다 = 도망 못 간다

    • 3차 시도 (느낌 기반):

  • "높은 곳에서 내려다본다" → 천리안? 마천루?
  • "Skyline이 지평선 아닌가?"
  • "지평선... 지평선 너머까지 쫓아간다"
  • "지평선 너머" → 글로벌 사격 느낌 + Lock은 효과 설명에서 전달

    • 결과: 순우리말 + 시적 + 직관적 + 글로벌 느낌 = 채택

    한자어 조합이 막히면, 의미를 분해하고 느낌부터 다시 접근한다

    케이스 스터디: "미리내 / 살별" (별 무기)

    원문: Stellar Staff / Astral Blade
    컨셉: 마법소녀 스타일 플라스틱 피규어

    1차 시도:

  • 성장(星杖) / 성검(星劍) → "무협 느낌, 마법소녀 아님" → 패스
  • 별빛 지팡이 / 별빛 검 → "괜찮은데... 뭔가 부족"

    • 2차 시도 (순우리말 탐색):

  • 미리내 = 은하수의 순우리말
  • 살별 = 혜성(꼬리별)의 순우리말

    • 적용:

  • 미리내 = 멀리서 빛나는 은하수 = 원거리 모드
  • 살별 = 꼬리 휘두르며 돌진하는 혜성 = 근접 모드

    • 결과: 순우리말 + 별 컨셉 + 마법소녀 감성 = 채택

    한자어가 안 맞으면 순우리말을 찾아본다. 의외의 보석이 있다.

    케이스 스터디: "물바다" (바다 Q)

    원문: Hydro Field
    문제: "물길"은 Field(넓은 장판)보다 좁은 통로 느낌

    후보:

  • 물판 → 직관적이나 평범
  • 물웅덩이 → 장난꾸러기 느낌, 귀여움
  • 물바다 → 캐릭터 이름이 "바다"인데?

    • 결과: "바다가 물바다 깔았어!" = 콜아웃 자연스러움 = 채택

    캐릭터 이름과 스킬명이 연결되면 기억하기 쉽고 콜아웃이 자연스럽다

    케이스 스터디: "하암~ / 마실" (미르 Q, W)

    원문: Dragon Breath, Sky Glide
    문제: "용의 숨결", "비상" — 너무 정석적. 게이머 소년 컨셉과 안 맞음.

    1차 시도 (품사 전환):

  • 명사+동사만 고집할 필요 있나?
  • 스몰더 W가 "엣취!"잖아 — 의성어/의태어는?
  • 용이 입 벌리고 뿜는 소리... "하암~"

    • 2차 시도 (마실):

  • "비상"은 너무 웅장함
  • 글로벌 이동인데 가볍게 "마실 갈게~" 하고 날아가는 느낌
  • 마실 = 이웃집에 놀러 가는 것 (순우리말)

    • 3차 시도 (무기명):

  • 잠자는 화산/자유로운 벼락 — 한자어인데 스킬은 의성어?
  • 하늘 무기처럼 형용사+명사 조합으로 통일
  • 잠자는 화산 / 자유로운 벼락
  • "잠자는"이 하암~(하품)과 연결됨

    • 결과: 게이머 소년 컨셉 + 장난스러운 톤 + 의외의 대비(마실 갈게~ → 급강하 쾅) = 채택

    품사를 바꿔보자. 명사/동사가 답이 아닐 수 있다.

    피해야 할 함정

    함정예시문제
    문화적 오해물대포일본어로는 장난감 물총, 한국어로는 시위 진압용
    과한 한자어창공쇄, 마천안읽기 어렵고 직관적이지 않음
